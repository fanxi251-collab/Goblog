##### 1、登录跳转问题

问题描述：登录表单提交后，页面没有自动跳转到文章编辑与发布的后台界面。

根本原因：
1. 路由顺序问题：原来的代码将所有路由放在同一个组里，登录页面被中间件拦截
2. 登录验证方式：之前使用的是 HTTP BasicAuth，不是表单登录

解决办法：
- 路由分离：公开路由（login/logout）和认证路由分开
- Cookie 存储：使用 c.SetCookie() 存储会话
- 中间件验证：AuthRequired() 检查 Cookie 是否存在
- 重定向：登录成功后使用 c.Redirect() 跳转

相关文件：
- internal/router/router.go - 路由配置
- internal/handler/admin/auth.go - 登录/登出处理
- web/templates/admin/login.html - 登录页面模板

##### 2、Dashboard 统计数据不显示

问题描述：后台仪表盘的文章总数、专栏数量、留言总数显示为空白。

根本原因：
1. 数据硬编码：dashboard.html 中数值硬编码为 0，未传递数据
2. 模板变量名大小写：Go 模板中 `.postsTotal` 应为 `.PostsTotal`（首字母大写）

解决办法：
- 路由中获取数据：postService.GetAll()、columnService.GetAll()、commentService.GetAll()
- 传递数据到模板：c.HTML() 时传入 gin.H{"PostsTotal": postsTotal, ...}
- 模板变量首字母大写：{{.PostsTotal}}

相关文件：
- internal/router/router.go - 路由配置中添加统计数据获取
- web/templates/admin/dashboard.html - 模板中使用大写变量名

##### 3、文章发布状态无法识别

问题描述：点击"直接发布"按钮后，前端提示"保存成功"，文章被保存到草稿箱而非发布。

根本原因：
- FormData 无法获取 submit 按钮的 value：HTML 表单中使用 `<button type="submit">` 时，FormData 无法获取按钮的 name/value
- 浏览器不会将提交按钮的值包含在 FormData 中

解决办法：
- 使用 onclick 绑定 JS 函数：`<button type="button" onclick="submitPost('publish')">`
- 手动添加 action：formData.append('action', action)
- 添加成功/失败提示：前端显示"发布成功！"或"保存成功！"

相关文件：
- web/templates/admin/post_edit.html - 修改按钮绑定方式和提交逻辑
- internal/handler/admin/post.go - 后端保存逻辑

##### 4、文章详情页专栏为空时报错

问题描述：访问首页文章（column_id=0）的详情页时，后端报错 "template: post.html:288:46: executing "post.html" at <.column.Slug>: nil pointer"

根本原因：
- 首页文章 column_id=0，后端尝试查找 ID=0 的专栏（不存在），返回 nil
- 模板访问 nil.Slug 时抛出空指针错误

解决办法：
- 只有 column_id > 0 时才获取专栏信息
- 如果 column 为 nil，创建空对象传递给模板避免报错

相关文件：
- internal/handler/front/column.go - Post 方法中添加 nil 检查

##### 11、文章正文图片不显示

问题描述：编辑文章时上传了图片，但文章详情页不显示图片。

根本原因：
1. 数据库存储的是 HTML 而非 Markdown：文章内容在数据库中已经是渲染后的 HTML，不是 Markdown 原文
2. 模板直接输出原始 HTML，未使用 safe 函数
3. Windows 路径问题：filepath.Join 生成反斜杠路径，导致 URL 编码错误
4. 上传返回 Markdown 格式链接，解析失败

解决办法：
- post.html 使用 `{{safe .post.Content}}` 直接输出 HTML
- 修改 file_service.go 返回路径时将反斜杠替换为正斜杠
- 修改 upload.go 返回 HTML img 标签而非 Markdown 格式

相关文件：
- web/templates/front/post.html - 使用 safe 函数输出 HTML
- internal/service/file_service.go - 路径正斜杠处理
- internal/handler/admin/upload.go - 返回 HTML img 标签

##### 12、后台导航栏动态路径问题

问题描述：后台页面之间跳转时链接错误，提示 404。例如从首页点击"文章"跳转到 `/admin/posts` 而非 `/admin-panel-7x9k2m3p/posts`。

根本原因：
1. 模板中硬编码了 `/admin/` 路径，而非使用动态的 `{{.adminPath}}`
2. router 未传递 adminPath 到模板
3. 浏览器可能缓存了旧的链接

解决办法：
- router.go 添加中间件设置 adminPath 到 context
- router.go 中每个 handler 传入 adminPath 参数
- 所有后台模板中 `href="/admin/"` 替换为 `href="{{.adminPath}}/"`
- 用户清除浏览器缓存或使用隐私模式测试

相关文件：
- internal/router/router.go - 添加 adminPath 中间件和参数传递
- web/templates/admin/*.html - 所有后台模板替换链接
